# Kiwi — User Flows

> Джерело правди: jtbd.md + sitemap.md  
> Flows — відображення логіки, не дизайн-рішення  
> Кожен вузол-екран існує в sitemap.md · стани (loading / empty / error) — не екрани  

**Легенда:**
| Форма | Значення |
|-------|----------|
| `[прямокутник]` | Екран або дія (з sitemap.md) |
| `{ромб}` | Точка рішення з гілками |
| `[/нахилений/]` | Стан: loading / empty / error |
| `([заокруглений])` | Термінальний стан: успіх або тупик |

## Main Job · Знайти готовий ресурс і отримати файл

**Job:** Коли я знаю що треба зробити але не хочу починати з нуля,  
я хочу знайти готовий ресурс що відповідає стилю і потребам проекту,  
щоб вийти на результат швидше, не жертвуючи якістю.

**Персона:** Куратор · primary · ✅ validated

```mermaid
flowchart TD
    START(["Куратор відкриває Kiwi"]) --> A1["A1 · Каталог"]

    A1 --> LOAD[/"Завантаження результатів"/]
    LOAD --> Q_NET{"Завантажилось?"}
    Q_NET -->|ні| ERR_NET[/"Помилка мережі"/]
    ERR_NET --> A1
    Q_NET -->|так| Q_RES{"Є результати?"}

    Q_RES -->|ні| EMPTY[/"Пусто — нічого не знайшлось"/]
    EMPTY -->|змінити запит або фільтр| A1
    EMPTY --> DEAD_EMPTY(["Кінець · Залишив без знахідки"])

    Q_RES -->|так| Q_COV{"Обкладинка відгукується?"}
    Q_COV -->|ні| Q_MORE{"Є ще картки?"}
    Q_MORE -->|так| Q_COV
    Q_MORE -->|ні| EXHAUST[/"Каталог вичерпано — підходящого немає"/]
    EXHAUST -->|розширити запит| A1
    EXHAUST --> DEAD_EX(["Кінець · Залишив без знахідки"])

    Q_COV -->|так| B1["B1 · Сторінка проекту"]
    B1 --> Q_FILE{"Екрани відповідають обкладинці?"}
    Q_FILE -->|ні| Q_BACK{"Спробувати іншу картку?"}
    Q_BACK -->|так| Q_COV
    Q_BACK -->|ні| DEAD_DIS(["Кінець · Розчарування після B1"])

    Q_FILE -->|так| Q_ACT{"Що далі?"}

    Q_ACT -->|"Зберегти ♥"| Q_AUTH_S{"Є акаунт?"}
    Q_AUTH_S -->|так| C1_S["C1 · Збережене"]
    Q_AUTH_S -->|ні| AUTH_S["E1 · Реєстрація / E2 · Вхід"]
    AUTH_S --> Q_REG_S{"Авторизація успішна?"}
    Q_REG_S -->|ні| DEAD_SAVE(["Кінець · Вийшов без збереження"])
    Q_REG_S -->|так| C1_S
    C1_S --> OK_SAVE(["Збережено · повернення до перегляду"])

    Q_ACT -->|Завантажити| D1["D1 · Гейт завантаження"]
    D1 --> Q_SUB{"Підписка активна?"}
    Q_SUB -->|так| DL_OK(["Файл завантажено · Main Job виконано"])

    Q_SUB -->|ні| Q_AUTH_D{"Є акаунт?"}
    Q_AUTH_D -->|так| D2["D2 · Вибір плану / Підписка"]
    Q_AUTH_D -->|ні| AUTH_D["E1 · Реєстрація / E2 · Вхід"]
    AUTH_D --> Q_REG_D{"Авторизація успішна?"}
    Q_REG_D -->|ні| DEAD_DL(["Кінець · Вийшов на гейті завантаження"])
    Q_REG_D -->|так| D2

    D2 --> Q_PLAN{"Тип оплати?"}
    Q_PLAN -->|підписка| STRIPE_S[/"Stripe · Checkout підписки"/]
    Q_PLAN -->|разова оплата| D3["D3 · Разова оплата"]
    D3 --> STRIPE_O[/"Stripe · Checkout разової"/]

    STRIPE_S --> Q_PAY{"Оплата успішна?"}
    STRIPE_O --> Q_PAY
    Q_PAY -->|так| DL_OK
    Q_PAY -->|відхилено| ERR_PAY[/"Помилка оплати — перевір картку"/]
    ERR_PAY -->|спробувати ще| D2
    ERR_PAY -->|скасувати| DEAD_PAY(["Кінець · Покупку скасовано"])
```

## RJ-1 · Знайти за стилем, а не категорією

**Job:** Коли я маю відчуття потрібної естетики але не слів для точного пошуку,  
я хочу знаходити ресурси за настроєм і візуальним характером,  
щоб не витрачати годину на підбір тегів і все одно прийти ні з чим.

**Персона:** Куратор · primary · ✅ validated · «Не міг знайти нудний стиль — за запитами їх просто не було»

```mermaid
flowchart TD
    START(["Куратор має відчуття стилю — але не знає точних слів"]) --> A1["A1 · Каталог"]

    A1 --> FILT["Обирає теги стилю або вводить запит настрою"]
    FILT --> LOAD[/"Завантаження результатів за стилем"/]
    LOAD --> Q_RES{"Є результати для цього стилю?"}

    Q_RES -->|ні| EMPTY[/"Пусто — стиль не представлений у каталозі"/]
    EMPTY --> Q_SYN{"Є схожі теги для спроби?"}
    Q_SYN -->|так| A1
    Q_SYN -->|ні| DEAD_STYLE(["Кінець · Стиль відсутній в каталозі — gap продукту"])

    Q_RES -->|так| Q_COV{"Обкладинка передає потрібний стиль?"}
    Q_COV -->|ні| Q_MORE{"Є ще картки?"}
    Q_MORE -->|так| Q_COV
    Q_MORE -->|ні| EXHAUST[/"Всі картки переглянуто — стиль не підходить"/]
    EXHAUST --> Q_WIDEN{"Розширити або змінити теги?"}
    Q_WIDEN -->|так| A1
    Q_WIDEN -->|ні| DEAD_EX(["Кінець · Підходящого стилю немає — залишив"])

    Q_COV -->|так| B1["B1 · Сторінка проекту"]
    B1 --> Q_CONF{"Реальні екрани підтверджують стиль?"}
    Q_CONF -->|ні — обкладинка обманула| Q_TRUST{"Продовжити пошук?"}
    Q_TRUST -->|так| Q_COV
    Q_TRUST -->|ні| DEAD_DIS(["Кінець · Розчарування — довіра до каталогу знизилась"])

    Q_CONF -->|так| OK(["RJ-1 виконано · стиль знайдено"])
```

## RJ-2 · Переконатись що файл відповідає обкладинці

**Job:** Коли я бачу обкладинку що зацікавила,  
я хочу побачити реальний вміст файлу до того як приймати рішення,  
щоб не витрачати час і емоції на розчарування після переходу.

**Персона:** Куратор · primary · ✅ validated · «Обкладинка виглядає добре, але при детальному розгляді дизайн посередній»

```mermaid
flowchart TD
    START(["Куратор клікнув на картку — обкладинка зацікавила"]) --> LOAD_B1

    LOAD_B1[/"Завантаження сторінки проекту"/] --> B1["B1 · Сторінка проекту"]

    B1 --> Q_HAS{"На сторінці є реальні екрани з файлу?"}

    Q_HAS -->|ні — тільки hero-shot| RISK[/"Ризик: рішення на основі одного зображення"/]
    RISK --> Q_TRUST{"Довіряєш лише обкладинці?"}
    Q_TRUST -->|ні — недостатньо| BACK1(["Повернення до A1 · Каталог — пропустив проект"])
    Q_TRUST -->|так — ризикую| Q_QUAL{"Якість деталей прийнятна?"}

    Q_HAS -->|так — є 2–5 реальних екранів| Q_MATCH{"Екрани відповідають очікуванням?"}
    Q_MATCH -->|ні — обкладинка обманула| Q_RETRY{"Спробувати іншу картку?"}
    Q_RETRY -->|так| BACK2(["Повернення до A1 · Каталог"])
    Q_RETRY -->|ні| DEAD(["Кінець · Розчарування — витрачено час і довіру"])

    Q_MATCH -->|так| Q_QUAL
    Q_QUAL -->|ні — посередньо| Q_FALLBACK{"Зберегти як запасний варіант?"}
    Q_FALLBACK -->|так| C1_FB["C1 · Збережене"]
    C1_FB --> SAVED_FB(["Збережено як запасний варіант · повернення до пошуку"])
    Q_FALLBACK -->|ні| BACK2

    Q_QUAL -->|так — це воно!| OK(["RJ-2 виконано · файл = обкладинка · переходить до дії"])
```

## RJ-3 · Зберегти що сподобалось без зайвих кроків

**Job:** Коли я знаходжу ресурс що може стати в нагоді пізніше,  
я хочу відкласти його в одне місце миттєво,  
щоб повернутись коли прийде відповідний проект, не шукаючи знову.

**Персона:** Куратор · primary · ✅ validated · «Або одразу зберігаю в обране, або одразу скачую»

```mermaid
flowchart TD
    START_A(["На A1 · Каталог — hover → іконка ♥ на картці"]) --> Q_AUTH{"Є акаунт?"}
    START_B(["На B1 · Сторінка проекту → кнопка Зберегти"]) --> Q_AUTH

    Q_AUTH -->|так| C1["C1 · Збережене"]
    C1 --> OK(["Збережено миттєво · RJ-3 виконано"])
    OK --> RESUME(["Продовжує перегляд A1 або B1"])

    Q_AUTH -->|ні| E1E2["E1 · Реєстрація / E2 · Вхід"]
    E1E2 --> Q_INTENT{"Готовий зареєструватись?"}
    Q_INTENT -->|ні — не очікував переривки| FAIL1(["Кінець · Вийшов не зберігши"])

    Q_INTENT -->|так| Q_REG{"Реєстрація успішна?"}
    Q_REG -->|ні| ERR[/"Помилка — перевір email або пароль"/]
    ERR --> Q_RETRY{"Спробувати ще?"}
    Q_RETRY -->|так| E1E2
    Q_RETRY -->|ні| FAIL2(["Кінець · Реєстрація не вдалась"])

    Q_REG -->|так| C1_POST["C1 · Збережене"]
    C1_POST --> OK2(["Збережено після реєстрації · RJ-3 виконано"])
    OK2 --> RESUME
```
