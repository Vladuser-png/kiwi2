# Kiwi — Product Brief

## Концепція
Kiwi — маркетплейс готових дизайн-ресурсів. Мета: скоротити час роботи дизайнера і дати швидкий вибір блоків, лендінгів, вебсайтів та інших UI-матеріалів.

## Аудиторія
18–50 років, дизайнери (junior → senior), фрілансери, агентства.

## Категорії ресурсів
| Категорія | MVP | Пріоритет |
|-----------|-----|-----------|
| Landing Page | ✅ | 1 |
| Website | ✅ | 1 |
| Шрифти | — | 2 |
| Презентації | — | 2 |
| Ілюстрації | — | 3 |
| Іконки | — | 3 |

## Контент-модель
- **Команда проекту** — внутрішня команда дизайнерів, наповнює платформу на старті
- **Зовнішні автори** — можуть завантажувати і продавати власні роботи

## Формати файлів
Платформа tool-agnostic: підтримуються всі формати — Figma, Sketch, Adobe XD, PowerPoint та інші. Файл завантажується напряму.

## Ліцензування
Покупка або завантаження файлу → автоматичні права на використання (стандартна комерційна ліцензія платформи). Без додаткових кроків.

## Бізнес-модель
- **Підписка** → необмежене завантаження всіх ресурсів
- **Разова оплата** → покупка окремого проекту за ціною автора

## Пошук і фільтрація
- **Головне:** пошук за темою / ключовим словом
- **Фільтри:** ціна, дата додавання, кольори, теги, інструмент (Figma / Sketch / XD / …)

## UX / Навігація
```
[Top Navbar: Landing Page | Website | (Шрифти) | (Презентації) | ...]
         ↓ клік на категорію
[Сторінка категорії]
  Пошук + Фільтри: ціна / дата / колір / тег / інструмент
  Картки з прев'ю → деталі → завантаження файлу
```

## Платформа
- **Фаза 1:** Desktop (пріоритет)
- **Фаза 2:** Mobile adaptive

## Команда
- Дизайнери проекту (на старті)

## Дедлайн
Не визначено — рухаємось за якістю.

## Диференціатор
Конкуренти: Envato Elements, UI8, Creative Market, Mobbin.

**Позиціонування:** *Kiwi — найшвидший спосіб знайти і використати готовий дизайн, в будь-якому інструменті, за одну підписку.*

- **Фокус на якість, а не обсяг** — менша, але відібрана база; команда тримає планку
- **Tool-agnostic** — не прив'язуємо до Figma чи будь-якого іншого інструменту
- **Підписка як Spotify** — одна ціна, доступ до всього

## MVP Scope
- [ ] Каталог: Landing Page + Website
- [ ] Пошук за ключовим словом
- [ ] Категорійна навігація + фільтри (ціна / дата / колір / тег / інструмент)
- [ ] Картка ресурсу + прев'ю
- [ ] Завантаження файлу (з підпискою / без)
- [ ] Реєстрація / логін (Supabase Auth)
- [ ] Підписка + разова оплата (Stripe)
- [ ] Кабінет автора — завантаження власних робіт
- [ ] Адмін: модерація контенту

## Правило роботи з файлами

**Уся інформація формується, зберігається і оновлюється виключно в .md файлах.**
HTML-файли слугують лише для відображення того, що є в .md — вони не є джерелом правди.

Порядок роботи:
1. Нова інформація → спочатку в .md
2. .html оновлюється з .md, а не навпаки
3. Якщо потрібно щось змінити — змінюй .md, потім синхронізуй .html

## Правило авто-пушу

**Після кожного git commit автоматично виконується git push.**
Реалізовано через `.git/hooks/post-commit`. Хук вже встановлений — нічого додатково налаштовувати не потрібно.

### Main Flow · Глибина тапів

**Куратор з активною підпискою:**
`START → A1 → (тап 1) B1 → (тап 2) Файл завантажено` = **2 тапи** ✅

**Без підписки:**
`START → A1 → B1 → D1 → D2 → Stripe → Файл` = 5 тапів

### Глобальна навігація

4 пункти + Logo = 5 елементів верхнього рівня:

| № | Пункт | Job |
|---|-------|-----|
| N1 | Landing Page | RJ-1 (категорія) |
| N2 | Website | RJ-1 (категорія) |
| N3 | Збережене ♥ | RJ-3 |
| N4 | Акаунт / Підписка | RJ-4, EJ-2 |

Максимальна глибина: ≤3 (A1 → B1 → D1 = рівень 3).
Primary flows: ≤2 тапи для підписника.

## Вайрфрейми

`wireframes/*.html` — 15 файлів, low-fi (сіра шкала, семантичний HTML, без кольору/тіней/радіусів/іконок — контракт у `wireframes/_conventions.md`).

**Покриття:** 7 екранів Ядра MVP (A1, B1, C1, D1, D2, E1, E2) з `wireframes/_screens.md` + розширення D3 (Разова оплата) і E3 (Особистий кабінет) з `wireframes/_screens.md` §Розширення. Кожен реальний стан (порожньо/помилка/завантаження/успіх) — окрема сторінка, тільки ✓-клітинки з матриці станів, нічого не вигадано понад flows.md.

**Навігація між файлами:** зліва на кожній сторінці — однакове `<nav class="wf-nav-tree">` (розділ → екран → стан), справжні `<a href>` між станами й екранами вздовж flows.md (Main Job), без глухих кутів.

**Свідомо поза скоупом:** RJ-3 (помилка форми E1/E2, порожній стан C1), D3/E3 вже включені; S1–S4 (сироти) — sitemap.md рекомендує їх видалити/backlog, вайрфреймів немає.

Аудит і виправлені дефекти — `wireframes/_critique.md`.
