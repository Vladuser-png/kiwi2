Розділ 04

Висновки для Kiwi MVP

Рішення по патерну карток

НАЙКРАЩЕ
Uniform Fixed Grid
ПРИЧИНА 1 · Формат

Лендінги — ландшафтний 16:9. Fixed grid = оптимальне використання простору.

ПРИЧИНА 2 · Порівняння

Превʼю → ціна → інструмент → автор. Рівна сітка дає точки кріплення.

ПРИЧИНА 3 · Швидкість

9 продуктів above the fold за 3 секунди. Головна ціль: скоротити час пошуку.

ДРУГИЙ ВИБІР: якщо є редакція
Editorial Hero + Grid

Якщо Kiwi запускає «Краще цього тижня» — 1–2 hero-картки + звична сітка. Комунікує кюрацію і довіру. Hero-слот = promoted placement для авторів.

НЕ ПІДХОДИТЬ
Masonry Grid

Скріни 16:9 → masonry нічого не додає. Ламає порівняння ціни. Сигналізує «натхнення», а не «купівля».

Топ-3 конверсійних механізми → MVP

01

Content-first hero без слів

Перший екран — сама сітка продуктів. Дизайнер розуміє цінність очима за 2 секунди. Джерело: Unsplash + Framer.

02

Lock-icon gate замість login wall

Весь каталог відкритий без реєстрації. Subscribe-момент — тільки при Download, коли вже є емоція. Джерело: Unsplash.

03

Масштаб числом у hero

«500+ landing pages · Figma, Sketch, XD» — лічильники під пошуком. Число продає підписку. Джерело: Mobbin.

Що НЕ спрацює

✕

Author PRO-badge як сигнал якості

Dribbble продає талант — PRO означає «варто найняти». Kiwi продає файл — «чи підійде мені цей шаблон?» Увага піде на соціальний статус замість якості контенту.

Замість: «Editor's Pick» від команди Kiwi — сигнал якості продукту, а не автора.

Gap ринку

Немає жодного продукту, який одночасно кюрований + tool-agnostic + підписка без лімітів + робочі файли (не скриншоти). Це і є вікно для Kiwi.

Main Job · Знайти готовий ресурс і отримати файл

Коли я знаю що треба зробити але не хочу починати з нуля, я хочу знайти готовий ресурс що відповідає стилю і потребам проекту, щоб вийти на результат швидше, не жертвуючи якістю.

RJ-1 · Знайти за стилем, а не категорією

Job Statement

Коли я маю відчуття потрібної естетики але не слів для точного пошуку, я хочу знаходити ресурси за настроєм і візуальним характером, щоб не витрачати годину на підбір тегів і все одно прийти ні з чим.

Персона: Куратор · primary · ✅ validated · «Не міг знайти нудний стиль — за запитами їх просто не було»

    START(["Куратор має відчуття стилю — але не знає точних слів"]) --> A1["A1 · Каталог"]

    A1 --> FILT["Обирає теги стилю або вводить запит настрою"]
    FILT --> LOAD[/"Завантаження результатів за стилем"/]
    LOAD --> Q_RES{"Є результати для цього стилю?"}

    Q_RES -->|ні| EMPTY[/"Пусто — стиль не представлений у каталозі"/]
    EMPTY --> Q_SYN{"Є схожі теги для спроби?"}
    Q_SYN -->|так| A1
    Q_SYN -->|ні| DEAD_STYLE(["Кінець · Стиль відсутній в каталозі — gap продукту"])

Jobs · тонке покриття — ризики реалізації

Жоден job не має Σ✓ = 0 — кожен закритий хоча б одним екраном. Але три мають одноточкове покриття або втрачають екран при відрізанні SIROTA.

RJ-2 · ОДНА ТОЧКА ВІДМОВИ · Σ✓ = 1
Файл = обкладинка
Тільки B1 покриває. Якщо реальні екрани з файлу (2–5) не реалізуються на B1 — job = 0 покриття. Зникає головний диференціатор від UI8.
Рішення · ЗАКРІПИТИ НА B1
«Екрани з файлу (2–5 реальних)» — must-have для B1, не nice-to-have. Без них RJ-2 = 0 і весь аргумент «ми не UI8» розвалюється. Перенести в вимоги до B1.
RJ-5 · ОДНА ТОЧКА ВІДМОВИ · Σ✓ = 1
Тримати руку на пульсі
Тільки A1 покриває — через сортування «Нові». Якщо сортування не реалізується — job = 0. Щоденний ритуал Куратора (Інтерв'ю #1) не підтримується.
Рішення · ЗАКРІПИТИ НА A1
Сортування «Нові надходження» — must-have для A1, не опція. Також критично для EJ-2 (підписка окупається). Retention залежить від цього.

Рішення · В BACKLOG разом з S4
До підтвердження SJ-2 через інтерв'ю з арт-директорами — share B1 є максимумом MVP. Після валідації: додати S4, оновити матрицю і flows.
