Unsplash

38

Еталон. Пошук у центрі героя, одразу масонрі-сітка. Unsplash+ виникає органічно через lock-іконки — не нав'язливо але очевидно. Нульова friction: перше «вау» ще до першого скролу.

Framer Templates

36

Герой — живі превʼю шаблонів без слова пояснення. "Use template" — один клік. Слабкість: мінімум соціальних доказів, немає лічильника завантажень.

UI8 · Mobbin

33

UI8: преміальний стиль, але підписка захована — головна CTA «Add to Cart». Mobbin: «300k+ screens» у заголовку — сильний масштаб, але контент за логіном — висока friction.

Dribbble

27

Антиприклад: два конкуруючих JTBD (натхнення + найм) на одній сторінці. Жодного чіткого «підпишись і отримай X». Не змішуй аудиторії на одному екрані.

Розділ 03

5 патернів розміщення карток

Принципово різні UX-підходи — не варіації. Кожен: як працює / де / коли підходить / коли ламається.

01 Masonry Grid змінна висота
Pinterest Behance Unsplash
Як працює Картки розміщуються у N колонок, висота кожної — за контентом. Немає фіксованих рядів і горизонтального вирівнювання. Коли підходить Контент з різними пропорціями (портрет/пейзаж/квадрат). Режим натхнення — не порівняння. Важлива щільність: показати максимум.
Працює

Фото, ілюстрації, мудборди — будь-який контент зі змінним aspect ratio.

Ламається

Ціна і CTA не вирівнюються між картками. При однаковому ratio — нічого не додає. Погано конвертує.

02 Uniform Fixed Grid рівна сітка
UI8 Framer Shopify
Як працює Всі картки однакового розміру, жорсткі колонки і ряди. Метадані (ціна, тег, автор) вирівняні під кожною карткою. Коли підходить Продукти порівнюються між собою. Стабільний aspect ratio контенту. Ціна і CTA критичні на картці.
Працює

Маркетплейси файлів, теми, шаблони. Швидке сканування по горизонталі: превʼю → ціна → інструмент.

Ламається

Контент з різними пропорціями — зрізається. Немає способу виділити featured без злому сітки.

03 Editorial Hero + Grid редакційна ієрархія
App Store Awwwards Webflow
Як працює 1–2 великі featured-картки (hero) → решта у стандартній сітці нижче. Розмір = пріоритет і кюрація. Коли підходить Є редакційна команда з editorial picks. Платформа хоче передати "ми відібрали найкраще". Підписна модель.
Працює

Передає довіру до платформи. Hero-слот — потенційно promoted placement для авторів.

Ламається

Без живої редакції hero виглядає випадковим. UGC: автори борються за featured-місце.

04 Netflix Row Model горизонтальні карусельні ряди
Netflix Spotify Envato
Як працює Контент розбитий по тематичних горизонтальних рядах (Landing Pages / Trending / New / By tool). Кожен ряд скролиться незалежно. Коли підходить Багато категорій із чіткими межами. Підписна модель — «тут є все». Ширина бібліотеки важлива.
Працює

Передає масштаб бібліотеки одним екраном. Ідеально для subscription upsell.

Ламається

Мала бібліотека — ряди напівпорожні. Одна категорія — надлишкова структура. Горизонтальний скрол ховає контент.

05 List View + Preview горизонтальний список
Gumroad B2B tools Doc libs
Як працює Кожен продукт — горизонтальний рядок: мінятюра ліворуч, праворуч — назва, ціна, інструмент, теги, автор, CTA. Все метадані без кліку. Коли підходить Метадані критичні для рішення (ціна/інструмент/ліцензія). B2B, купівля за специфікацією.
Працює

Максимум інформації без переходу в деталі. Ідеально для порівняння 3+ атрибутів одночасно.

Ламається

Візуальний продукт: різко зменшує площу превʼю. Режим browsing — погано підходить для натхнення.

Розділ 04

Висновки для Kiwi MVP

Патерн карток для MVP
НАЙКРАЩЕ
Uniform Fixed Grid

Лендінги — ландшафтний скрін 16:9. Fixed grid використовує простір оптимально і дозволяє вирівняти ціну + інструмент + автора. 9 продуктів above the fold за 3 секунди.

ДРУГИЙ ВИБІР: якщо є редакція
Editorial Hero + Grid

Якщо Kiwi запускає регулярні добірки «Краще цього тижня» — 1–2 hero-картки + звична сітка. Комунікує кюрацію і довіру. Hero-слот = promoted placement для авторів.

НЕ ПІДХОДИТЬ
Masonry Grid

Скріни лендінгів мають однаковий ratio — masonry нічого не додає. Ламає порівняння: ціна першої картки вища за сусідню на 40px. Сигналізує «натхнення», а не «купівля».

Топ-3 конверсійних механізми → MVP
01

Content-first hero без слів

Перший екран — сама сітка продуктів, без банера і пояснень. Дизайнер розуміє цінність очима за 2 секунди. Джерело: Unsplash + Framer.

02

Lock-icon gate замість login wall

Весь каталог відкритий без реєстрації. Subscribe-момент — тільки при натисканні Download, коли вже є емоція. Джерело: Unsplash.

03

Масштаб числом у hero

«500+ landing pages · Figma, Sketch, XD» — лічильники під пошуком. Число продає підписку краще за будь-який копірайт. Джерело: Mobbin.

Що НЕ спрацює
✕

Author PRO-badge як сигнал якості

Dribbble-патерн: бейдж «PRO» поруч з іменем автора на кожній картці. Dribbble продає талант — badge означає «варто найняти». Kiwi продає файл — покупцю байдуже, чи автор PRO чи ні. Питання одне: «чи підійде мені цей шаблон?»

Ризик: увага піде на соціальний статус замість якості контенту. Шум без конверсійного ефекту.

Замість: кюраторська позначка від редакції Kiwi — «Editor's Pick» або «Top Rated» — це сигнал якості продукту, а не статусу автора.

Gap ринку

Немає жодного продукту, який одночасно кюрований + tool-agnostic + підписка без лімітів + робочі файли для використання (не скриншоти для натхнення). Це і є вікно для Kiwi.
